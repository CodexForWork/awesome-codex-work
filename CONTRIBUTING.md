# Contributing to Awesome Codex Work

Thank you for helping make this a practical, source-backed directory instead of another loose link list.

## What a Good Contribution Includes

A useful contribution should include:

- A primary or otherwise verifiable source link.
- One honest sentence describing what the source actually says or enables.
- The work problem it helps with.
- The human review step a professional should run before relying on the output.
- A date, version, or access note when the entry describes something that changes.
- A clear directory category or playbook location.

## Claims That Need Sources

Any claim about capability, availability, pricing, access, performance, adoption, model behavior, supported platforms, or security posture needs a source a reader can check.

If you cannot source it, say less.

## What Will Be Rejected

- Unsourced claims about performance, benchmarks, user numbers, pricing, access, or availability.
- Anything framed as a way to bypass, replace, avoid paying for, or get free access to OpenAI or Codex.
- OpenAI logos, official-style visuals, or anything implying endorsement, sponsorship, certification, partnership, or approval.
- Private workspace details, internal notes, customer or employer names, credentials, logs, screenshots, or confidential material.
- Marketing language without a source behind it.
- Duplicates that do not add a distinct, verifiable detail.
- Community posts, comments, forum text, or blog prose copied into this repo instead of summarized and linked.

## Pull Request Shape

Open a pull request with:

- The entry or document change.
- Source link or links.
- Retrieval date.
- A short explanation of why it belongs in a Codex-powered work directory.
- Any known limits, such as preview status, plan restrictions, platform scope, or version-specific behavior.

Maintainers check traceability first, wording second. If a claim cannot be traced, the pull request will be asked to cut or rewrite it before merge.

## Independence and Brand Boundary

CodexWork is independent and unofficial. Do not submit wording that implies OpenAI endorsement, sponsorship, certification, partnership, or approval.

Do not add OpenAI logos or official-style visuals. Descriptive references to Codex, ChatGPT, OpenAI, MCP, and related tools should stay factual and sourced.
