# Source Traceability Checklist

Use this when Codex writes a memo, guide, research summary, directory entry, review note, or public-facing explanation.

## Claim Inventory

Mark each factual claim as:

- sourced;
- assumption;
- inference;
- opinion;
- unsupported and should be removed.

## Source Quality

Prefer:

- official documentation;
- official repositories;
- standards/specification repositories;
- maintained source repositories;
- primary announcements;
- direct data exports you are allowed to use.

Use caution with:

- community posts;
- comments;
- forum threads;
- marketing pages;
- old blog posts;
- copied snippets without context.

## Trace Format

For each entry or document, include:

- source title;
- URL;
- publisher or repository owner;
- retrieved date;
- what the source supports;
- what it does not support;
- freshness risk.

## Red Flags

- A fluent paragraph with no links.
- "Studies show" without a study.
- "Latest" without a date.
- Plan/pricing/access claims without an official source.
- A copied community quote without rights review.
- A source that supports a weaker claim than the draft makes.

## Practical Rule

If a claim would matter to a manager, customer, lawyer, finance lead, security lead, or public reader, trace it or cut it.
