# Human Review Before Use

Use this checklist before relying on Codex output in professional work.

## 1. Source Check

- What sources did Codex use?
- Are the sources linked or otherwise recoverable?
- Are source dates visible?
- Are claims separated from assumptions?
- Did Codex use any source it was not allowed to use?

## 2. Scope Check

- Did the output answer the stated goal?
- Did it follow the constraints?
- Did it add unrelated work?
- Did it skip any required section, file, test, or deliverable?

## 3. Accuracy Check

- Are names, dates, numbers, versions, and links correct?
- Are time-sensitive claims current?
- Are unsupported claims removed or marked?
- Does the recommendation follow from the evidence?

## 4. Privacy Check

- Does the output include private names, emails, paths, file names, source excerpts, customer data, employer data, credentials, logs, screenshots, or internal notes?
- Are public examples safely abstracted?
- Is the destination allowed to receive this information?

## 5. Action Check

- Did Codex only draft, or did it also change files, call tools, run commands, post, send, upload, deploy, or merge?
- Are side effects listed?
- Is there a diff, log, change summary, or audit trail?

## 6. Done Check

- What did "done" mean in the prompt?
- Is the done condition actually satisfied?
- Were tests, checks, or manual verification run?
- If checks were not run, is the blocker stated clearly?

## 7. Human Decision

Pick one:

- Use as-is after review.
- Use after edits.
- Send back to Codex with specific corrections.
- Do not use.

## Minimal Approval Note

For high-impact work, save:

- output path or link;
- source list;
- review date;
- reviewer;
- decision;
- remaining risks.
