# Start Here

Awesome Codex Work has two jobs:

1. Help you find credible Codex-related resources.
2. Help you turn those resources into work that a human can review.

The repo is intentionally not a generic AI-tools list. A resource belongs here when it has a meaningful relationship to Codex or Codex-style work and helps people produce reviewable work products.

## The First Question

Ask this before choosing any tool or template:

> What work product am I trying to create, and what would a human need to check before using it?

Examples:

- "I need a weekly manager update from project notes."
- "I need a research-to-decision memo with source links."
- "I need Codex to inspect a repo and draft a safe pull request."
- "I need an agent to use a browser, but only inside a specific test account."
- "I need a repeatable instruction that preserves source boundaries."

Once the work product is clear, choose a surface, frame the task, and define review.

## How to Use the Directory

The Directory answers:

- What is the source?
- What work problem does it help with?
- What can a reader verify?
- What should remain human-reviewed?
- What privacy, security, or approval boundary applies?

Do not treat directory entries as endorsements. Treat them as starting points for evaluation.

## How to Use the Playbook

The Playbook gives reusable work patterns:

- task framing;
- durable guidance;
- review checklists;
- source traceability;
- MCP and connected-tool safety;
- turning output into deliverables.

Use the Playbook before you ask Codex for high-impact work. It is cheaper to define the review boundary up front than to repair an untraceable output later.

## Minimum Review Rule

Before using Codex output professionally, answer:

- What sources did it use?
- What did it assume?
- What did it skip?
- What changed in files, apps, tools, or external systems?
- What should a human still decide?

If you cannot answer those questions, the output is not ready.
